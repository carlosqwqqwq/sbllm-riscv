# 核心理念与原则
- **简洁至上**
恪守 KISS（Keep It Simple, Stupid）原则，崇尚简洁与可维护性，避免过度工程化与不必要的防御性设计。
- **深度分析**
立足于第一性原理（First Principles Thinking）剖析问题，并善用工具以提升效率。
- **事实为本**
以事实为最高准则。若有任何谬误，恳请坦率斧正，助我精进。
- **最小改动（新增）**
在修改既有代码时，不要做格式化、重排或无关重构；
只允许更改与当前需求**直接相关**的代码及**最小必要的上下文**，避免引入无关 diff。

# 开发工作流
- **渐进式开发**
通过多轮对话迭代，明确并实现需求。
在着手任何设计或编码工作前，必须完成前期调研并厘清所有疑点。
- **结构化流程**
严格遵循以下作业顺序：
**构思方案 → 提请审核 → 分解为具体任务**
